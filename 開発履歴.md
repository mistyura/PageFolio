## v0.9.8.1 — 2026-04-13（ビューアー強化・UI改善）

### 追加機能

#### 閲覧モード / 編集モード 切替
- ヘッダーに **👁 閲覧モード / 📝 編集モード** トグルボタンを追加（F5 ショートカット対応）
- **閲覧モード（デフォルト）**: 右ツールパネルを非表示にし、プレビューを最大化。PDFビューアーとして使いやすいレイアウト
- **編集モード**: 右ツールパネルを表示し、全機能を利用可能
- モード状態は `pagefolio_settings.json` に保存し、次回起動時も引き継ぎ

#### ウィンドウ状態の引き継ぎ
- 前回終了時のウィンドウ **位置・サイズ** を設定ファイルに保存し、次回起動時に復元
- 編集モード時のサッシ位置（左パネル幅・右パネル幅）も保存・復元

### UI 改善

#### 起動時レイアウト比率の調整
- デフォルト（閲覧モード）: サムネイル 15% ／ プレビュー 85%（右パネル非表示）
- 編集モード: サムネイル 15% ／ プレビュー 62% ／ ツール 23%
- ヘッダー高さを `font_size × 5.5`（最小60px）に拡大

#### プレビューツールバーのボタン表示改善
- ツールバー高さを `font_size × 4.5`（最小52px）に拡大（従来44px）
- ナビゲーションボタン幅を `width=4` に拡大し、左右パディングを増加
- ページ番号ラベルに固定幅（`width=8`）を設定し、ページ数が変わっても崩れないように
- ズームラベルにも固定幅（`width=6`）を設定

## v0.9.3 — プラグイン読み込み時の ModuleNotFoundError 修正

### バグ修正
- **プラグイン読み込み時の `ModuleNotFoundError: No module named 'pdf_editor'` を修正**
  - **症状**: `plugins/page_info.py` 読み込み時に `ModuleNotFoundError: No module named 'pdf_editor'` が発生
  - **原因**: プラグインが `from pdf_editor import PDFEditorPlugin` としているが、メインファイルは `pagefolio.py` であり `pdf_editor.py` というファイルは存在しない
  - **対応**: `load_plugin` 内でプラグインモジュールを実行する直前に、`sys.modules["pdf_editor"]` に `pagefolio` 自身（`__main__` または `pagefolio` として登録済みのモジュール）を登録。既存プラグインコードを変更せずに解決

### 内部変更
- `load_plugin` メソッド内で `spec.loader.exec_module()` 実行前に `sys.modules["pdf_editor"]` へ `pagefolio` モジュール自身を登録
