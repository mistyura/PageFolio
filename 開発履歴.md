## v1.2.1 — 2026-05-26（UI 改善）

### 変更内容

OCR ダイアログを「準備中…」のまま自動実行する仕様から、ユーザーが内容を確認してから「読み取り実行」ボタンで開始する仕様に変更。サーバ URL（参照のみ・設定メニュー連動）とモデル選択 ComboBox をダイアログに直接配置。結果テキストエリアをクリアするボタンも追加。

- **OCR ダイアログ**:
  - サーバ URL を読み取り専用 Entry で表示（参照のみ・設定メニューで設定した値を表示）
  - モデル選択 ComboBox を追加。「📥 モデル一覧取得」ボタンで LM Studio から `/v1/models` を取得して反映。ここで選んだモデルはこの実行のみ反映（設定の永続値は変更されない）
  - 「▶ 読み取り実行」ボタン（Accent スタイル）を追加。押下時に OCR を開始し、ボタンは無効化される
  - 「🧹 クリア」ボタンを追加。結果テキストエリア・進行表示・実行状態（`_started`/`_done`/`_cancel_flag`）をリセットし、再度「読み取り実行」できるようにする（実行中は無効）
  - 自動開始（`self.after(50, self._start_worker)`）を廃止
  - キャンセルボタンの挙動を統一: 実行前/完了済みなら閉じる、実行中ならキャンセル要求
  - 既定サイズを 880×624（minsize 820×560）に拡張し、6 ボタン（クリア/コピー/保存/読み取り実行/キャンセル/閉じる）が全て表示されるようにした
- **`LLMConfigDialog`（新規）**:
  - URL / モデル / OCR 解像度倍率 / タイムアウト / モデル一覧取得 / 接続テスト / 適用 / キャンセル
  - OCR ダイアログと設定ダイアログ双方から呼び出して使う共通画面
- **設定ダイアログ**:
  - 旧 LM Studio セクション（インライン）を「🔍 LLM 設定…」ボタンに置換。`LLMConfigDialog` を開いて編集する
  - 高さを縮小（旧 `fs * 42` → `fs * 24`）

## v1.2.0 — 2026-05-26（機能追加）

### 追加機能

#### LM Studio Vision API による OCR

- ローカル動作の LM Studio（OpenAI 互換 API）のビジョンモデルを使い、PDF ページから文字を抽出する機能を追加
- **追加依存なし**: `urllib.request`（標準ライブラリ）のみで実装（`requests` 不使用）
- **対象選択**:
  - `🔍 現在ページを OCR`: プレビュー表示中のページ 1 枚
  - `🔍 選択ページを OCR`: サムネイルで複数選択中のページを一括処理
- **プロンプトプリセット**: 「素のテキスト抽出 / 表を Markdown / 全体を Markdown」の 3 種類をダイアログで切替
- **OCR ダイアログ**:
  - ttk.Progressbar による進行表示（`N/M ページ処理中…`）
  - スクロール可能な Text ウィジェットに `--- Page N ---` 区切り付きで結果を逐次表示
  - クリップボードへコピー / `.txt`・`.md` ファイル保存
  - `threading.Event` によるキャンセル機構（実行中の閉じる操作にも確認ダイアログ）
- **エラー処理の分岐**:
  - `ConnectionError`（LM Studio 未起動等）/ `TimeoutError`: バッチ全体を中止
  - `RuntimeError`（HTTP エラー・Vision 非対応モデル等）: 当該ページのみスキップして続行
- **設定**: 設定ダイアログから URL・モデル・解像度倍率（1.0〜4.0）・タイムアウト（10〜600 秒）を変更可能。「モデル一覧を取得」「接続テスト」ボタン付き
- **永続化**: `lm_studio_url` / `lm_studio_model` / `ocr_prompt_preset` / `ocr_scale` / `ocr_timeout` を `pagefolio_settings.json` に保存

## v1.1.2-4 — 2026-05-26（配布形式変更）

### 変更内容

#### ビルド形式を `--onefile` から `--onedir` に変更

- **動機**: onefile は起動時に一時ディレクトリへ自己展開する仕組みのため、初回起動が遅く、Windows Defender / SmartScreen で誤検知されるリスクがあった
- **対応**: PyInstaller のオプションを `--onefile` から `--onedir` に変更。`PageFolio.exe` と `_internal/` ディレクトリを一式として配布する形式に切り替え
- **配布物**: GitHub Release には `PageFolio-v1.1.2-4-onedir.zip`（約36MB）を添付。展開後に `PageFolio.exe` を実行する運用となる
- **アプリ自体の機能変更なし**（`APP_VERSION` は `v1.1.2` のまま）

### 変更ファイル

|----------|----------|
| ビルドコマンド | `pyinstaller --onefile ...` を `pyinstaller --onedir ...` に変更 |

## v1.1.2 — 2026-05-19（バグ修正）

### 修正内容

#### PNG 単体読み込み時にエラーダイアログが表示される問題を修正

- **症状**: PNG 等の画像ファイルを1枚だけ選択して開くと、エラーダイアログが表示される
- **原因**: `_open_pdf_path` が `fitz.open(path)` で画像ファイルを画像ドキュメントとして開いていた。その後 `_show_preview` のメインスレッド処理（`viewer.py` 69行目）で `self.doc.tobytes()` を呼ぶが、画像ドキュメントは PDF 形式でないため失敗し、エラーダイアログが表示されていた。複数ファイル選択時（`_do_open_merged`）は既に `_open_path_as_pdf()` で PDF 変換していたため問題なし。単体選択時のみ漏れがあった。
- **対応**: `_open_pdf_path` でも `_open_path_as_pdf()` を使うよう変更。画像を `convert_to_pdf()` で PDF バイト列に変換してから開くことで `tobytes()` が正常に動作するよう修正。

| ファイル | 変更内容 |
|----------|----------|
| `pagefolio/file_ops.py` | `_open_pdf_path` 内の `fitz.open(path)` を `self._open_path_as_pdf(path)` に変更（1行） |
| `CLAUDE.md` | バージョン v1.1.2 に更新 |

## v1.1.1 — 2026-05-19（バグ修正）

### 修正内容

#### 画像ファイル複数読み込み時に結合失敗する問題を修正

- **症状**: PNG 等の画像ファイルを複数選択して開くと、結合されずにエラー終了する
- **原因**: `fitz.open(image_path)` で開いた画像 Document を `insert_pdf()` に渡していたが、`insert_pdf()` は PDF Document のみ受け付けるため失敗していた
- **対応**:
  - `FileOpsMixin` に `_open_path_as_pdf(path)` ヘルパーを追加
  - 画像ファイルは `fitz.Document.convert_to_pdf()` で一旦 PDF バイト列に変換してから `fitz.open(stream=...)` で再オープンする
  - `_do_open_merged`（複数ファイルを開く）と `_do_insert`（挿入操作）の両方に適用

### 変更ファイル

| ファイル | 変更内容 |
|----------|----------|
| `pagefolio/file_ops.py` | `_open_path_as_pdf()` ヘルパー追加、`_do_open_merged` で使用 |
| `pagefolio/page_ops.py` | `_do_insert` で `_open_path_as_pdf()` を使用 |

## v1.1.0 — 2026-05-15（機能追加）

### 追加機能

#### ページ結合・リサイズ（サイズ結合）
- 選択した複数ページを 1 枚の大きなページに結合する機能を追加
- 例: A4 縦×2 ページを横並びで結合 → 1 枚の A3 横ページとして出力
- 配置方向は **横並び（左→右）/ 縦並び（上→下）** をダイアログで選択可能
- 出力サイズは選択ページの合計サイズで自動算出（横並び: 幅の合計 × 高さの最大、縦並び: 幅の最大 × 高さの合計）
- 異なるサイズのページ混在にも対応（高さ/幅は最大値を採用）
- `fitz.Page.show_pdf_page()` で各ページを所定の矩形にそのまま貼り付け
- 元ページは結合ページで置き換え、選択は新ページに移動
- Undo はスナップショット形式（`pdf_bytes`）で保存

#### ファイルを閉じる（アプリは終了しない）
- ファイルセクションに「📕 ファイルを閉じる」ボタンを追加
- 確認ダイアログで誤操作を防止し、未保存変更を破棄して空状態に戻る
- Undo/Redo 履歴・サムネイルキャッシュ・選択状態を全てクリア
- `on_file_close` プラグインイベントを発火

#### プレビューとサムネイルの同期バグ修正（viewer.py）
- **症状**: ページ削除・回転後、左ペイン（サムネイル）と中央ペイン（プレビュー）の表示が一致しない
- **原因**: `_show_preview()` のバックグラウンドスレッドが `filepath` 有りの場合にディスク上の元ファイルを再オープンしていた。メモリ上の編集状態（削除・回転等）が反映されず、保存前の古い内容が表示されていた
- **対応**: 常に `self.doc.tobytes()` でメモリ上のドキュメントをシリアライズし、`stream=` で開くように統一。`filepath` 分岐を削除

  - **対応**: `load_plugin` 内でプラグインモジュールを実行する直前に、`sys.modules["pdf_editor"]` に `pagefolio` 自身（`__main__` または `pagefolio` として登録済みのモジュール）を登録。既存プラグインコードを変更せずに解決

### 内部変更
- `load_plugin` メソッド内で `spec.loader.exec_module()` 実行前に `sys.modules["pdf_editor"]` へ `pagefolio` モジュール自身を登録
