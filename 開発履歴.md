# PDF Editor 開発履歴

---

## v1.0.0 — 初回リリース

### 追加機能
- PDF ファイルの読み込み・保存（上書き／名前を付けて保存）
- ページサムネイル一覧表示
- プレビュー表示（拡大・縮小・スクロール）
- ページ回転（90° / 180° / 270°）— 複数ページ対応
- ページ削除 — 複数ページ対応
- トリミング（上下左右の余白を数値入力で削減）
- 別 PDF からページ挿入（単一ファイル）
- PDF 結合（単一ファイル、末尾に追加）
- ページ移動（番号指定）
- Ctrl+クリックによる複数ページ選択
- 全選択 / 選択解除ボタン
- 起動用バッチファイル（ライブラリ自動インストール付き）

---

## v1.1.0 — 結合機能強化

### 追加・変更機能
- **複数ファイル結合対応**: 結合・挿入で複数 PDF を同時選択可能
- **結合順ダイアログ新設**: 選択ファイルの順番を GUI で並び替え可能
  - ▲▼ ボタンによる順番変更
  - リストからの個別除外
  - 結合後の合計ページ数をリアルタイム表示
- 挿入機能も複数ファイル対応（選択順に指定位置へ順次挿入）

---

## v1.2.0 — トリミング UI 刷新

### 追加・変更機能
- **ドラッグ選択トリミング**: プレビュー上でマウスドラッグして残す範囲を直接指定
  - 選択範囲外を暗幕で表示
  - 選択中の座標・サイズ（pt）をリアルタイム表示
  - 範囲リセットボタン追加
- 数値入力によるトリミング UI を廃止

---

## v1.3.0 — ページ並び替え強化・終了ボタン追加

### 追加・変更機能
- **サムネイルのドラッグ＆ドロップによるページ入れ替え**
  - ドラッグ中にゴースト（半透明プレビュー）を表示
  - 挿入位置を赤い点線インジケーターで表示
  - ドラッグしない場合は従来通りクリック選択として動作
- **終了ボタン追加**（ファイルを開いている場合は確認ダイアログあり）
- 起動バッチをシンプル化（`cd /d "%~dp0"` で同フォルダ起動に統一）

---

## v1.4.0 — パフォーマンス改善・安全性強化・Undo対応

### バグ修正
- **WM_DELETE_WINDOW 未設定**: ウィンドウ「×」ボタンで閉じた場合も確認ダイアログを表示し `doc.close()` を実行するように修正
- **上書き保存の二重例外処理漏れ**: incremental 保存失敗時のフォールバック保存が更に失敗した場合のエラーハンドリングを追加
- **全ページ削除防止**: 全ページを選択して削除しようとした場合に警告を表示し、最低1ページを維持するよう制御
- **D&D バウンダリチェック強化**: サムネイルパネル外へドロップした場合の挿入先計算を安全に処理

### パフォーマンス改善
- **サムネイルキャッシュ化**: `thumb_cache` 辞書を導入し、ページ変更がないサムネイルの再レンダリングを回避
- **選択変更時のプレビュー再描画抑止**: `_refresh_thumbs_selection_only` 新設
- **トリミング暗幕の描画最適化**: `canvas.coords()` による座標更新方式に変更
- **MergeOrderDialog のページ数キャッシュ**: 初回のみ取得してキャッシュに変更

### 追加機能
- **Undo / Redo（Ctrl+Z / Ctrl+Y）**: 最大20回まで取り消し・やり直し可能
- **キーボードショートカット**: Ctrl+O / S / Shift+S / Z / Y、Delete
- **ファイル D&D 対応**: `windnd` ライブラリ使用（未インストール時はサイレントスキップ）

### UI 改善
- 上書き保存に確認ダイアログ追加
- トリミングモードの視覚強調（赤背景＋🔴アイコン）
- 挿入位置ダイアログの説明文改善
- 結合ダイアログにページ数表示（例: `report.pdf (12p)`）
- ボタンにショートカットキー表記追加

### コード品質
- 未使用引数の削除
- 終了時の `doc.close()` 確実化

## v1.6.0 — UX改善・トリミングバグ修正

### バグ修正
- **トリミング保存時の `CropBox not in MediaBox` エラーを修正**: CropBox 座標を丸め処理し、MediaBox 内に微小マージン（0.01pt）付きでクランプするように変更。エラー時のフォールバックメッセージも追加

### 追加・変更機能
- **ファイル未開時のボタングレーアウト対応**: PDF 未開時にファイル操作・ページ操作・トリミング・挿入結合のボタンをすべて `disabled` 化。ファイルを開くと自動的に `normal` に復帰
- **空状態プレビューに案内文表示**: ドキュメント未開時にプレビュー領域中央へ「📂 ファイルを開いてください (Ctrl+O)」を表示
- **「範囲未選択」テキストの文字色変更**: トリミングセクションの「範囲未選択」ラベルの色を `SUCCESS`（緑）から `TEXT_SUB`（グレー）に変更し、未選択状態が目立ちすぎないように調整

## v1.7.0 — UIレビュー課題完了・レイアウト改善

### UI 改善
- **Undo/Redo ボタンを横並び化**: 2ボタンを1行に収め、右ペインの省スペース化。回転ボタン（↺ ↕ ↻）が初期表示でスクロールなしで見えるように改善
- **左パネル幅リサイズ対応**: 左ペイン（サムネイル一覧）と中央ペイン（プレビュー）の境界を `PanedWindow` に変更。ドラッグで左パネル幅を自由に調整可能に（最小150px）
- **右ペイン初期スクロール位置の確実なリセット**: `after_idle` + `after(100)` + `after(300)` の3段階で `yview_moveto(0)` を呼び出し、環境差による初期スクロールのズレを解消

### その他
- UIレビュー（2026-03-12）の全課題が対応完了

## v1.8.0 — 設定画面追加・テーマ切替・フォントサイズ変更

### 追加機能
- **設定ダイアログ新設**: 右ペイン最下部の「⚙ テーマ・フォント設定…」ボタンから設定画面を開く
- **テーマ切替**: ダーク / ライト / システム設定に合わせる の3モードをラジオボタンで選択
  - 「システム設定に合わせる」はWindowsのレジストリ（`AppsUseLightTheme`）から自動判定
  - ライトテーマはダークテーマと同じレイアウトで明るい配色を採用
- **フォントサイズ変更**: 8〜16pt をスピンボックスで選択（リアルタイムプレビュー付き）
- **設定の永続化**: `pdf_editor_settings.json` に保存し、次回起動時も設定を維持

### 内部変更
- カラーテーマをグローバル定数から `THEMES["dark"]` / `THEMES["light"]` 辞書に移行
- 全UIコンポーネントが `C["BG_DARK"]` 等のテーマ変数を参照する構成に変更
- フォントサイズは `self._font(delta)` ヘルパーメソッドで相対指定（ベースサイズ + delta）
- テーマ/フォント変更時は `_rebuild_ui()` でUI全体を再構築
- Undo/Redo ボタンテキストを「↩ Ctrl+Z」「↪ Ctrl+Y」に短縮
- 回転ボタンを2行構成に変更（上段: 左90°・右90°、下段: 180°）

- **デフォルトフォントサイズを12ptに変更**: 初期値を10pt→12ptに。既存設定ファイルがある場合はそちらを優先

## v1.8.2 — レイアウト最適化・プレビュー視認性改善

### UI 改善
- **右ペインを固定幅に変更**: PanedWindow から分離し、ウィンドウ拡大時は中央プレビューのみが広がるレイアウトに。右ペイン幅はフォントサイズに応じて自動計算（`font_size × 22`、最小260px）
- **ライトテーマのプレビュー背景色を調整**: `#ffffff` → `#c8c8d0`（グレー）に変更し、白いPDFページとの境界を明確化
- **ページのドロップシャドウ表示**: プレビュー上のPDFページに影（右下3pxオフセット）と1px枠線を追加。ダーク・ライト両テーマで視認性向上
- **回転ボタンを2行構成に変更**: 上段「↺ 左90°」「↻ 右90°」、下段「↕ 180°」で右ペイン幅でも確実に全ボタン表示
- **Undo/Redo ボタンテキスト短縮**: 「↩ Ctrl+Z」「↪ Ctrl+Y」に短縮し横並びでも見切れを防止
- **ヘッダー高さのフォントサイズ連動**: `font_size × 5`（最小56px）で自動調整
- **ポップアップ（拡大表示）にもドロップシャドウ適用**

---

## v1.8.3 — ダイアログのフォントサイズ対応

### UI 改善
- **結合順ダイアログのサイズをフォントサイズに連動**: 幅 `font_size × 40`（最小480px）、高さはファイル数に応じて動的拡大。ファイルが5つ以上の場合は1ファイルあたり約 `font_size × 2.5` px ずつ高さが増加
- **結合順ダイアログをリサイズ可能に変更**: ウィンドウ枠をドラッグで拡大縮小可能（最小400×350px）
- **ファイルリストの表示行数を動的調整**: ファイル数+2行（最小6行、最大20行）で Listbox の高さを設定
- **設定ダイアログのサイズをフォントサイズに連動**: 幅 `font_size × 32`（最小380px）、高さ `font_size × 24`（最小280px）
