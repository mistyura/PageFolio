# S02: バックグラウンドレンダリング — UAT

**Milestone:** M001
**Written:** 2026-05-04

## UAT Type

- UAT mode: human-experience
- Why this mode is sufficient: バックグラウンドスレッドによる UI 応答性改善は Tkinter のメインループと密結合しており、ヘッドレス環境では検証不可。ruff + pytest による静的・単体検証は完了済みのため、残る確認は実際の GUI 操作による体感確認のみ。

## Preconditions

- PageFolio が `python pagefolio.py` で起動できること
- テスト用の大規模 PDF（50ページ以上推奨）が手元にあること
- 小規模 PDF（数ページ）も用意しておくこと

## Smoke Test

大規模 PDF を開いてページをクリックし、1秒以内にプレビューが更新されることを確認する。

## Test Cases

### 1. 大規模 PDF のサムネイル即時表示

1. 50ページ以上の PDF をファイルメニュー「開く」から開く
2. サムネイルパネルの表示を観察する
3. **Expected:** サムネイルが全件レンダリングされる前でも画面がフリーズせず、プレースホルダーが表示されたあと順次サムネイル画像が読み込まれること

### 2. ページ切替時の UI 応答性

1. 50ページ以上の PDF を開く
2. サムネイルをクリックして異なるページに高速で切り替える（5回以上）
3. **Expected:** クリックのたびに UI が応答し、ヘッダーのページ番号が更新されること。「固まった」感覚がないこと

### 3. ズーム変更時の UI 応答性

1. PDF を開く
2. ズームスライダーを素早く動かす
3. **Expected:** ズーム変更中も UI が応答し続け、最終的なズーム値でプレビューが更新されること

### 4. ドキュメント切替時のレンダリングキャンセル

1. 大規模 PDF を開く（サムネイル読み込み中）
2. すぐに別の PDF を開く（または同じファイルを再度開く）
3. **Expected:** 前の PDF のサムネイルが新しい PDF のウィジェットに誤描画されないこと。新しい PDF のサムネイルが正しく表示されること

### 5. Undo/Redo 後のプレビュー正常更新

1. PDF を開いてページ操作（回転・削除など）を実行する
2. Ctrl+Z で Undo する
3. **Expected:** Undo 後のプレビューとサムネイルが正しく更新されること（前の doc のレンダリング結果が残留しないこと）

## Edge Cases

### 未保存結合 doc のプレビュー

1. 複数の PDF を結合する
2. 保存前の状態でページを切り替える
3. **Expected:** 結合後の doc のプレビューが正しく表示されること（filepath=None でもクラッシュしないこと）

### 高速ページクリックでのスレッド競合

1. PDF を開く
2. サムネイルを非常に高速にクリックして複数ページを切り替える
3. **Expected:** 最終的に選択されたページのプレビューが表示されること。古いページのプレビューが誤表示されないこと

## Failure Signals

- UI が数秒以上フリーズする（サムネイル読み込み中）
- 前の PDF のサムネイル画像が新しい PDF のウィジェットに残留する
- `python pagefolio.py` 実行時のコンソールに Exception トレースバックが出力される
- プレビュー領域が空白のまま更新されない

## Not Proven By This UAT

- ヘッドレス環境での自動化テスト（Tkinter のメインループ依存のため不可）
- 超大規模 PDF（500ページ以上）での実際のメモリ使用量と応答時間の計測
- スレッドリーク（デーモンスレッドのため、プロセス終了時に自動回収されるが長時間使用での蓄積は未確認）
- PyMuPDF 将来バージョンでのスレッド安全性（現バージョン固定済み）

## Notes for Tester

- サムネイルが「ちらつく」挙動（プレースホルダー → 画像への切り替え）は仕様通り。フリーズではない。
- コンソール出力に `DEBUG:pagefolio.viewer:` で始まるログが出ても問題なし（stale チェックの診断ログ）。
- Windows 11 で Tkinter のスレッドコールバック（after(0,...)）は通常 10〜50ms 遅延する。
