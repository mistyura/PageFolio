# PageFolio — テスト方針と実行方法 (Testing)

このプロジェクトでは、テスト容易性の高い「純ロジック層」の分離設計に加え、UI 結合テスト、セキュリティ上の漏洩防御スキャン、耐久ストレス試験などを網羅したテストスイートを構築しています。

## テストツールとフレームワーク

- **テストフレームワーク**: `pytest` (`9.1.1`)
- **カバレッジツール**: `pytest-cov` (`7.1.0`)
- **インポート設定**: `pyproject.toml` にてテストラン用の pythonpath に `src` が指定されており、Ruff で `S101` (assert 警告) がテストコードに対して除外されています。

---

## 主なテストファイルと責務

テストコードはすべて `tests/` 配下に格納されています。主なテストは以下の通りです。

| テストファイル | テスト内容・責務 |
| :--- | :--- |
| `conftest.py` | テスト用の共通フィクスチャ（モック Tk アプリ、一時ディレクトリ、モック OCR プロバイダなど） |
| `test_imports.py` | パッケージ全体のインポートテスト。古いインポートパスが破壊されていないかどうかの後方互換チェック |
| `test_lang_parity.py` | `ja`/`en` 間の言語辞書キーの一致、および引数のプレースホルダー数の整合性を検証 |
| `test_pagination.py` | 境界値（0ページ、巨大ページ数、マイナスインデックス等）における窓計算の数学的正確性を検証 |
| `test_ocr_pipeline.py` | スレッドセーフな `PipelineState` や、リトライ・サーキットブレーカーの純ロジックをモック API で網羅的に検証 |
| `test_undo_stress.py` | 120ページのPDFを用いてUndo/Redoを連続で実行し、Blobが正しくディスクに退避され、メモリリークやファイルの物理削除漏れ（不変条件）が発生しないか検証する耐久試験 |
| `test_settings_keyguard.py` | 設定ファイル `pagefolio_settings.json` に機密キー（APIキー）が絶対に書き込まれないことを検証 |
| `test_source_keyguard.py` | リポジトリのソースコード内に誤って実際の API キーパターン（sk-proj-...等）がハードコードされていないかを自動スキャンする防御テスト |
| `test_pdf_ops.py` | PyMuPDF を利用したページの回転、削除、結合、トリミング、暗号化などのPDF加工処理の結合テスト |

---

## テストの実行方法

開発中は、コードのコミットや Pull Request の作成前に必ずすべてのテストが通過することを確認します。

### 1. 全テストの実行
仮想環境を有効にした状態で、プロジェクトのルートディレクトリから実行します。
```bash
pytest
```

### 2. 特定のテストファイルの実行
```bash
pytest tests/test_undo_stress.py
```

### 3. カバレッジレポートの生成
```bash
pytest --cov=pagefolio tests/
```
