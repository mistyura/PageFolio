<objective>
OCRDialog 側の旧セッションキー入力 UI（_key_frame / api_key_var）とヘルパー（_needs_session_key / _ensure_cloud_session_key）を撤去し（V171-KEY-03・導線を LLMConfigDialog へ一元化）、クラウド実行前に鍵の解決可否のみを確認して未解決なら明示エラーを出す軽量な _check_cloud_api_key へ置換する（成功基準2）。この置換により RunPod キーが claude スロットへ誤格納される旧2分岐バグ（Pitfall 1）が構造的に消滅する（V171-KEY-04）。撤去に伴い不要化する LANG キーと依存テストも同一プランで整理する。

Purpose: 「鍵の収集」と「鍵の存在確認」が同居していた旧関数を分解し、入力 UI は LLMConfigDialog（Plan 02）に一元化した上で、OCRDialog は解決可否の確認とエラー橋渡しだけを担う。単純削除では成功基準2の「明示的なエラー」が失われる（Pitfall 2）ため、_resolve_api_key を呼ぶだけの確認関数へ確実に置換する。
Output: _check_cloud_api_key・旧 UI/ヘルパー撤去・_open_llm_config への配線・後継テスト・不要 LANG キー削除。
</objective>
