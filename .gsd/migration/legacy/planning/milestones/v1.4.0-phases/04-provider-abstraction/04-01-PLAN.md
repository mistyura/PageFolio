## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | `OCRAPIKeyError` / Provider 属性 | accept | 本プランでは APIキー秘密情報を一切扱わない。`OCRAPIKeyError` は定義のみ（env_var 名のみ保持、キー値は保持しない）。クラウド Provider・キー流入は Phase 5（V14-D-02）。`ocr_providers.py` に `os.environ` 読み出しを追加しない |
| T-04-02 | Spoofing / SSRF | `LMStudioProvider.ocr_image` / `list_models` の urllib 接続先 | accept | 接続先 URL は既存 settings（`lm_studio_url`、既定 localhost:1234）由来をコンストラクタ経由で踏襲。本プランで新規外部接続先・ユーザー任意 URL の追加はなし。`# noqa: S310`（urllib 直叩き）は既存方針 V14-D-01 を継承 |
| T-04-03 | Information Disclosure | 例外メッセージ / logger 出力 | mitigate | 例外メッセージは現行同様 `body[:500]` で切り詰め、APIキー等の秘密を含めない。`logger` に settings 辞書や base64 画像本体をダンプしない（Pitfall Security Mistakes 準拠） |

</threat_model>
