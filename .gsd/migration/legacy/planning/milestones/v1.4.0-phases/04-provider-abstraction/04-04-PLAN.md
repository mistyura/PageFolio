## LMStudioProvider コンストラクタ仕様（CR-02 の単一情報源）

`pagefolio/ocr_providers.py:76` の `LMStudioProvider.__init__` シグネチャ（CR-02 で `self.provider` 再生成に使う確定パラメータ）:

```
LMStudioProvider(url, model, timeout=120, max_tokens=-1, temperature=0.1)
```

| 引数 | 型 | CR-02 での供給元 |
|------|----|----------------|
| `url` | str | `self.url_var.get().strip()`（UI のサーバ URL 欄。readonly だが値は保持される） |
| `model` | str | `self.model_var.get().strip()`（ダイアログのモデル Combobox の live 値） |
| `timeout` | int | `_on_run` でクランプ済みの `self._effective_timeout`（= `self._ocr_timeout`） |
| `max_tokens` | int | `self.max_tokens_var.get()`（live 値・クランプ推奨） |
| `temperature` | float | `self.temperature_var.get()`（live 値・クランプ推奨） |

`LMStudioProvider` は `OCRProvider` のサブクラスなので、再生成後も `run_parallel(self.provider, ...)` がそのまま動く（`provider.ocr_image` / `provider.max_concurrency` のみ参照される）。`default_concurrency` / `max_concurrency` はクラス属性なのでインスタンス再生成で失われない。
