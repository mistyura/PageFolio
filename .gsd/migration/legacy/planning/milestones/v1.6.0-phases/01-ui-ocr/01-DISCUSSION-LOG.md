## 二重化の解消方式（V16-UI-01）

| Option | Description | Selected |
|--------|-------------|----------|
| 読み取り専用表示 | 現在適用値をグレーアウト表示。編集は「⚙ LLM 設定…」に一元化。透明性が高い | ✓ |
| 完全撤去 | OCR 画面から数値 UI を削除。最もスッキリだが現在値が見えない | |

**User's choice:** 読み取り専用表示（推奨）

### 反映タイミング

| Option | Description | Selected |
|--------|-------------|----------|
| 即時反映 | LLM 設定で OK/適用直後に OCR 画面の表示も更新（既存 `_apply_llm_settings` 活用） | ✓ |
| 次回開き直し時 | ダイアログを閉じて開き直すと反映。実装簡単だが一息違和感 | |

**User's choice:** 即時反映（推奨）
**Notes:** OCR 画面に既存の「⚙ LLM 設定…」ボタンがあり、ライブ更新経路を再利用できる。

## 実行時オプションの線引き（V16-UI-01）

| Option | Description | Selected |
|--------|-------------|----------|
| OCR 画面に編集可能で残す | プロンプトプリセット/埋め込み無視/セッションキーは実行毎に変える値。一元化対象外 | ✓ |
| 一部も LLM 設定へ移す | プリセット等も LLM 設定に集約。OCR 画面はさらにシンプルだが実行毎の切替が手間 | |

**User's choice:** OCR 画面に編集可能で残す（推奨）
**Notes:** 一元化対象は「永続的な数値パラメータ」に限定すると確定。

## スライダーの実問題と配置（V16-UI-02）

### 実問題の特定

| Option | Description | Selected |
|--------|-------------|----------|
| 縮小時に細くなり使えない | ボタンと同一行で幅を奪い合い、狭ペインで潰れる | ✓ |
| 別の場所にあり隠れる | 実機ではペイン下部などに見えて隠れる（コードと実装ずれ） | |
| 位置より見た目/操作性 | 位置は良いがラベル無し・倍率不明等の改善が主眼 | |

**User's choice:** 縮小時に細くなり使えない

### 目標配置

| Option | Description | Selected |
|--------|-------------|----------|
| ボタンの下の独立行に全幅配置 | 全選択/解除行の下に専用行を新設し `fill="x"` 全幅。幅を競合させない | ✓ |
| 「ページ一覧」ラベル行の右へ | hdr 側へ移動。ただし横並びのため縮小時の幅問題が残りうる | |
| スライダーにラベル/アイコンも付ける | 独立行＋サイズアイコンや倍率表示で操作性向上 | |

**User's choice:** ボタンの下の独立行に全幅配置（推奨）
**Notes:** スライダーの機能（範囲 0.5〜2.5・コールバック）は不変、配置のみ変更。

## Claude's Discretion

- 読み取り専用化の具体手段（`state="readonly"` / `disabled` / ラベル置換）は実装時判断。「現在値が読める」ことは必須。
- スライダー独立行への倍率表示/アイコン追加は任意（要件外装飾）。
- url/model（LM Studio 固有）の表示出し分けはプロバイダ選択に応じて実装裁量。
