# Requirements: Milestone v1.6.0 品質向上・AI強化・設定/UI改善

> **Milestone goal:** 体感品質（回転プレビュー即時反映・エラーハンドリング UX）と AI 出力品質（Markdown 整形・プロバイダ別プロンプト最適化）を底上げし、設定の二重化を解消して大量ページ対応で UI を整える。
> **出典:** `.planning/NEXT-MILESTONE-HANDOFF.md`（統合ロードマップ + 2026-06-18 追記の仕様要望 S1〜S3）
> **REQ-ID 形式:** `V16-[CATEGORY]-[NUMBER]`

## v1.6.0 Requirements

### 設定 / UI 改善（UI）

- [x] **V16-UI-01** (S1🔴): OCR パラメータ（タイムアウト・並列度・リトライ等）を「LLM設定」画面のみで設定でき、OCR テキスト抽出画面の重複入力 UI が廃止または読み取り専用化されている。設定値は一意に解決され「どちらが効くか分からない」二重化が解消されている。
- [x] **V16-UI-02** (S2🟡): サムネイルサイズ変更スライダーが、左ペインを縮小しても常に見える位置（「全選択／解除」ボタン付近または「ページ一覧」ラベル横）に配置され、機能は従来どおり動作する。
- [x] **V16-UI-03** (S3🟡): 大量ページの PDF で、サムネイル一覧を可変件数（既定 20 ページ単位）ごとに区切って表示でき、表示件数が `pagefolio_settings.json` に永続化される。ページング下でも D&D 並び替え・複数選択が全ページインデックスと整合して正しく動作する。

### 品質向上（QUALITY・プランA）

- [ ] **V16-QUAL-01** (H1中): ページを回転すると、回転状態がプレビュー表示に即時反映される（再読込やページ切替を待たずに見た目が更新される）。
- [ ] **V16-QUAL-02** (H2低): API キーが設定ファイル・ソースコード・ログに平文で露出していないことを監査で再確認し、`_SENSITIVE_KEYS` ガード・環境変数/セッションメモリ限定の挙動が回帰テストで担保されている。
- [ ] **V16-QUAL-03** (H5低): max_tokens クランプ（H-1）と 429/レート制限リトライ（H-6）が実 API（または実機相当の検証手順）で期待どおり動作することを検証し、結果が記録されている。
- [ ] **V16-QUAL-04** (M1低〜中): レート制限・トークン超過・応答途切れ時に、ユーザーが状況と次のアクションを理解できるエラーメッセージ／UX が提示される（指数バックオフ・Retry-After クランプ・サーキットブレーカーの残細部を磨く）。

### AI 強化（AI・プランC）

- [ ] **V16-AI-01** (M3中): OCR 結果ビューア（`OCRDialog`）で、`markdown` プリセット出力が整形表示され、プレーンテキストよりも見出し・箇条書き等が読みやすく提示される。
- [ ] **V16-AI-02** (M4中): プロバイダ別にプロンプトテンプレートが最適化され（Claude=XML タグ構造 / Gemini=明示指示）、OCR 精度・出力品質が引き出される。既存のカスタムプロンプト機構（v1.5.0）と両立する。

## Future Requirements（次マイルストーン以降）

- **L1**: 検索可能 PDF 化（OCR テキストの透明レイヤー埋込）— 価値高だが重い
- **L2**: 印刷機能（OS 印刷連携）
- **L3**: 画像（ロゴ）透かし対応（現状テキスト透かしのみ）
- **L4**: OCR プロバイダ追加（OpenAI / Azure 等の標準同梱）
- **M2**: OCR 結果の永続化／履歴（セッション跨ぎ保存）— プランB 側、本マイルストーン非選択
- **H4**: PDF パスワード解除対応 — プランB 側、本マイルストーン非選択
- **M5**: API キー入力 UX 改善（OS キーチェーン保存等）
- **M6**: ショートカット設定 GUI（現状 JSON 直接編集のみ）

## Out of Scope（本マイルストーン明示除外）

- **OAuth 接続（Claude / Gemini）** — 正規 API が非対応・配布バイナリに client secret を安全に埋め込めない・ローカル完結アプリでバックエンドプロキシ方式と不一致（`NEXT-MILESTONE-HANDOFF.md` §1 確定事項）
- **ストリーミング表示（SSE）** — OCR はページ単位バッチ処理で設計不一致（並列実行＋進捗バーで体感担保済）
- **マルチモーダル画像入力の新機能化** — 既に OCR で base64 画像送信を実装済み
- **L5**: 残コメント / "off" UI 整理 — 軽微・別タスク
