## 実施内容

### 1. 機能追加: ページ→画像変換（1ページ1ファイル）

- **目的**: LLM（claude.ai / ChatGPT 等のチャット UI）にページ画像を渡してテキストを
  読み取らせる用途。内蔵 OCR と異なり API キー不要でアプリ外に持ち出せる汎用出口。
- 右パネルに「🖼 画像変換」セクションを新設（閲覧モードでも使用可）。
- `ExportImagesDialog`（新規 `pagefolio/dialogs/export_images.py`）:
  対象ページ（全/選択/範囲指定）・長辺ピクセル数プリセット
  （1024 / **1568=Claude 推奨** / 2048 / **3072=Gemini 推奨**）・PNG/JPEG（品質 10〜100）。
- レンダリングは `page.rect` 基準のため**トリミング・回転の編集結果が画像に反映**される。
- ファイル名は分割機能と同じ `{元名}_p{ゼロ詰め番号}.{拡張子}`、上書き確認を再利用。
- 実装: `pagefolio/page_ops.py` に `export_page_image` / `compute_export_scale` /
  `parse_page_ranges`（モジュール関数化）、定数は `constants.py`。

### 2. バグ修正: 「縮小して保存」で元ファイルへ上書きできない

- **原因**: `fitz.Document` が元ファイルのハンドルを保持しており、同一パスへの
  非インクリメンタル `save()` は拒否され、Windows ではロックで `os.replace` も失敗。
- **対応**: `_overwrite_current_file()` — ① `doc.tobytes()` でメモリへシリアライズ →
  ② `doc.close()` でハンドル解放 → ③ tmp 経由 `os.replace` → ④ 開き直し。
  失敗時は bytes から doc を復元して継続。`_save_file` のフォールバックも同方式化。
  縮小保存ダイアログに現ファイル名を初期表示。

### 3. 機能追加: OCR リラン / リスタート（続きから再実行）

- **背景**: クラウド OCR が途中で致命的エラー（タイムアウト等）になると全結果を失い
  最初からやり直しだった（実例: 18 ページ中 12 ページ成功後にタイムアウト）。
- **リラン**: 実行終了後（完了/エラー/キャンセル）に「▶ 読み取り実行」を自動再有効化。
  クリア不要で全体再実行（押下時に前回結果を破棄）。
- **リスタート**: 「⏯ 続きから再実行」ボタン新設。成功済み結果を保持したまま
  **エラー/未処理ページのみ** API へ再送信し、完了時に全ページ分をページ順統合表示。
  コスト確認は未処理ページ数で見積（`_confirm_cost(page_count)`）。
- 設計判断: リランの出力は**クリアして再出力を維持**（追記方式はコピー/保存との
  整合性が崩れるため不採用）。前回結果が必要な場合は再実行前に手動保存する運用。

### 4. 機能追加: サーキットブレーカー + エラー件数つき完了表示

- リトライ上限到達が **3 ページ連続**（`CB_CONSECUTIVE_FAILURES`）で実行を中断
  （`kind="circuit_breaker"`）。成功で連続カウンタはリセット。中断後も成功分は保持され
  「続きから再実行」で再開できる案内を表示。
- エラーページが残ったまま完了した場合は
  「⚠ 完了: {count}/{total} ページ成功・エラー {err}ページ」を**警告色**で表示。

## セッション中の知見（運用ノウハウ）

- gemma 系の HTTP 500 は**サーバ側要因**で日によって変動（昨日は頻発・今日は安定）。
  API 仕様変更ではなく共有サービング基盤の負荷変動と推定。
- **解像度 4.0 は逆効果**: ペイロード肥大（面積 16 倍）でアップロード時間・TPM 消費が
  増えタイムアウトを誘発。10 ページ超で詰まったのは無料枠 TPM 枯渇と整合的。
  クラウド OCR は **1.5〜2.0 推奨**。
- OCR の結果欄は全ページ完了後に一括描画（実行中に空欄なのは仕様）。
- リスタートの統合表示は「表示=コピー/保存内容」の整合性を保つための仕様
  （API 送信は未処理ページのみ。コスト確認・進捗バーの母数で確認可能）。
