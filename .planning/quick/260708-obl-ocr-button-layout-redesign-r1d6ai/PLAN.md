# Plan: OCR ダイアログのボタン群を右ペインへ再配置（v1.7.3）

> 実装は同日完了 — 実施結果は [SUMMARY.md](./SUMMARY.md) を参照。

## 背景

ユーザー報告: 「OCRテキスト抽出画面で、初期状態で開いたときに『続きから再実行』ボタンが
隠れてしまう。隠れてしまうボタンがあるのはよくない。一律下にあるボタン群を、メインと
同じ右ペインへもっていきたい。」

`OCRDialog`（`pagefolio/ocr_dialog.py`）はボタン 8 個（コピー/保存/サマリ/クリア +
再開/実行/キャンセル/閉じる）を 1 つの `btn_row` に `pack(side="left")` /
`pack(side="right")` で横一列に配置していた。ウィンドウ幅がボタン合計幅（フォント
サイズ・言語・OS のボタン余白に依存）を下回ると、Tkinter の pack は後からパックされた
ウィジェットから優先的に切り詰めるため、最後にパックされる「⏯ 続きから再実行」から
順に隠れる構造だった。`_center` の初期幅計算（`max(1150, fs * 90)`）では環境差
（DPI スケーリング・フォント実測幅）を吸収しきれない。

## 対応方針

1. **右ペイン化**: ボタン群を下部の横一列から、結果テキスト右側の縦型右ペインへ移動する。
   メイン画面の右パネル（`ui_builder._build_tools`）と同じ「セクション見出し
   （`BG_CARD` + `WARNING` 色ラベル）+ 全幅ボタン（`fill="x"`）」の構成に揃える。
2. **セクション分割**: 機能グループで分ける。
   - 「▶ 実行」: 読み取り実行（`Accent.TButton`）/ 続きから再実行 / キャンセル（`Danger.TButton`）
   - 「📋 結果」: クリップボードにコピー / テキストファイルに保存 / サマリ作成 / クリア
   - 「✕ 閉じる」: セクション外・右ペイン最下部に `pack(side="bottom")` で固定（誤操作分離）
3. **構造的解消**: 縦積みの全幅ボタンにすることで、ウィンドウ幅がボタン表示に影響しない
   構造となり「幅不足でボタンが隠れる」問題自体を排除する。
4. **状態遷移は不変**: ボタンの属性名（`run_btn` / `resume_btn` / `cancel_btn` /
   `copy_btn` / `save_btn` / `summary_btn` / `clear_btn` / `close_btn`）と
   有効/無効の状態遷移ロジックには手を入れない（レイアウトのみの変更）。
5. **i18n**: セクション見出しの LANG キー（`ocr_sec_run` / `ocr_sec_result`）を
   ja / en 両方へ追加し、キー数パリティを維持する。

## 想定される変更ファイル

- `pagefolio/ocr_dialog.py`（本体・`_build` のボタン行 → 右ペイン化）
- `pagefolio/lang.py`（セクション見出しキー追加）
- `pagefolio/constants.py`（`APP_VERSION`）
- `README.md`（バージョンバッジ）
- `開発履歴.md`（変更履歴追記）

## 検証方法

- `ruff check . && ruff format .` / `pytest` 全件グリーン。
- headless（Xvfb）環境で `OCRDialog` を実際に構築し、初期サイズ（1150x680）と
  最小サイズ（960x620）の両方で全 8 ボタンの矩形（`winfo_rootx/rooty` + 幅/高さ）が
  ダイアログの可視領域内に収まることを座標検証する自動チェックスクリプトで確認する
  （実機の GUI 目視確認は環境上できないため代替手段とする）。
- スクリーンショットを取得し、新レイアウトの見た目（セクション構成・ボタン配置）を確認する。
