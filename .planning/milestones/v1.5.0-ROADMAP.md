## Phase Details

### Phase 1: PDF ページ操作・編集機能の拡充

**Goal**: PDF編集ツールとしての基本機能を底上げし、PDFの操作の幅を広げる
**Requirements**: V15-PAGE-01, V15-PAGE-02, V15-PAGE-03
**Success Criteria** (what must be TRUE):
  1. ページを指定してその後ろに白紙ページを挿入できる
  2. ページにテキスト透かし・ページ番号を追加できる
  3. ページ操作（削除、分割、結合）時にTOCが保持される

### Phase 2: UI / UX とパフォーマンスの改善

**Goal**: ユーザーがより直感的に操作できるUIと快適なUXを提供する
**Requirements**: V15-UIUX-01, V15-UIUX-02, V15-UIUX-03
**Success Criteria** (what must be TRUE):
  1. スライダーを使ってサムネイルサイズを動的に変更できる
  2. 外部PDFをサムネイルペインにドラッグ＆ドロップして指定位置に挿入できる
  3. JSON設定ファイルからショートカットキーをカスタマイズできる

### Phase 3: AI・OCR連携のさらなる進化

**Goal**: AIの強みを活かし、OCRを用いた自由度の高いデータ抽出を可能にする
**Requirements**: V15-OCR-01, V15-OCR-02
**Success Criteria** (what must be TRUE):
  1. 設定ダイアログでカスタムプロンプトを入力・保存できる
  2. カスタムプロンプトがOCRバックエンドに正しく渡されて処理される

### Phase 4: テスト・品質保証

**Goal**: 新機能の追加によって既存の挙動が壊れていないことを保証する
**Requirements**: V15-QA-01, V15-QA-02
**Success Criteria** (what must be TRUE):
  1. `ruff check .` と `ruff format .` を通過する
  2. `pytest` による自動テストを通過する
