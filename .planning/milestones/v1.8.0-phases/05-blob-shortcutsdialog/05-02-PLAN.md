    _build_thumbnails を優先度付き2段レンダリングへ改修する。現行どおり _thumb_gen をインクリメントして gen を採り、窓 [lo, hi) のプレースホルダを全生成する（ウィジェットは窓内全生成を維持・D-01）。次に _visible_local_range() から可視ローカル区間を得て、可視 global 区間 (vis_lo_g, vis_hi_g) = (lo + vis_lo_local, lo + vis_hi_local) を作り、pagination.prioritized_render_order(lo, hi, vis_lo_g, vis_hi_g) で描画順序リストを得る。順序リストの可視分（先頭 vis_hi_g - vis_lo_g 件）は root.after(0) 連鎖で先に描画し、残り（先読み分）は root.after_idle 連鎖で描画する（D-03 アイドル先読み・総仕事量は現行と同じ）。render_next 相当の各描画で既存の世代ガード条件 `self._thumb_gen != gen or not self.doc` を先頭に保持し、_get_thumb_photo と placeholder_labels[i - lo] への configure を行う（プレースホルダ参照は窓ローカル添字 i - lo のまま）。

    tests/test_viewer.py に、prioritized_render_order を使った描画順序の期待（可視先頭・残り後続）を検証するロジックテストと、_render_visible_thumbs の世代ガード（gen 不一致で No-op = _get_thumb_photo を呼ばない）を types.SimpleNamespace スタブ + _get_thumb_photo をスパイに差し替えた形（Tk root 不要な範囲）で検証するテストを追加する。実 Canvas レイアウトを要する部分は _visible_local_range のフォールバック分岐（frames 空→(0,0)）のみ純粋に検証する。
