## テンプレート保存モデル

| Option | Description | Selected |
|--------|-------------|----------|
| ペア保存 | 1テンプレート = カスタム+サマリの組。UIは1つのテンプレート選択欄で両方切り替わる | ✓ |
| 別々の一覧 | カスタム用テンプレート一覧とサマリ用テンプレート一覧を独立管理 | |

**User's choice:** ペア保存（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| settings.json 内辞書 | 既存の _load_settings/_save_settings パターンを延長 | ✓ |
| 専用ディレクトリの個別mdファイル | prompt_templates/*.md のように1テンプレート=1ファイル | |

**User's choice:** settings.json 内辞書（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| 削除を禁止する | アクティブなテンプレートは削除ボタンを無効化するか切替を促す | ✓ |
| 削除して「テンプレートなし」に戻す | 削除後はカスタム/サマリ欄が空（デフォルトプロンプト使用）に戻る | |

**User's choice:** 削除を禁止する（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| 保存時に重複拒否 | ShortcutsDialog の重複拒否パターンを踏襲 | ✓ |
| 同名で上書き保存を許可 | 同じ名前で保存すると既存テンプレートを上書き | |

**User's choice:** 保存時に重複拒否（推奨）

**Notes:** 4問とも推奨オプションが選択された。続行確認では「次の領域へ」を選択。

## フォールバック実行時の挙動

| Option | Description | Selected |
|--------|-------------|----------|
| 未処理ページのみ再開 | 既存のresume/_pending_pages()仕組みを流用。成功済みページは保持 | ✓ |
| 全ページを新プロバイダで再実行 | プロバイダごとにOCR品質が異なる可能性を考慮し全てやり直す | |

**User's choice:** 未処理ページのみ再開（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| 3番目以降も都度確認しつつ提案 | 設定されたチェーンを最後まで順に辿り、各段で送信先確認ダイアログを再提示 | ✓ |
| フォールバックは1回限り | 最初の候補への切替提案のみ。それも失敗したらユーザーが手動で別プロバイダを選ぶ | |

**User's choice:** 3番目以降も都度確認しつつ提案（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| fatalエラー全般（APIキー未設定含む） | PipelineState.fatal_msgが確定する全ケースでフォールバックを提案 | ✓ |
| 接続系エラーのみ（APIキー未設定は除外） | APIキー未設定は別途明示エラー表示のままにする | |

**User's choice:** fatalエラー全般（APIキー未設定含む・推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| OCRと同様に適用 | _confirm_summary_cost等既存のサマリ専用確認経路を流用 | ✓ |
| OCRのみ（サマリは非対象） | サマリ生成失敗時は従来通りエラー表示のみでフォールバックは提案しない | |

**User's choice:** OCRと同様に適用（推奨）

**Notes:** 4問とも推奨オプションが選択された。続行確認では「次の領域へ」を選択。

## フォールバック順設定UI

| Option | Description | Selected |
|--------|-------------|----------|
| リスト+上へ/下へボタン | MergeOrderDialog（merge.py）のtk.Listbox+「上へ」/「下へ」ボタンの既存パターンを流用 | ✓ |
| ドラッグ&ドロップ | サムネイルD&D（dnd.py）のような直感的なドラッグ並び替え | |

**User's choice:** リスト+上へ/下へボタン（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| 全プロバイダを一覧に含める | APIキー未設定でも一覧に表示し、実行時にAPIキー未設定エラーとして次候補へ進む | ✓ |
| 設定済みプロバイダのみ表示 | APIキー等が解決可能なプロバイダのみ候補一覧に表示 | |

**User's choice:** 全プロバイダを一覧に含める（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| 新規独立セクション | 「🔁 フォールバック」として既存のプロバイダ選択セクションの直後に独立配置 | ✓ |
| プロバイダ選択セクション内に統合 | 既存の「OCRプロバイダ選択」セクション内にフォールバック設定を埋め込む | |

**User's choice:** 新規独立セクション（推奨）

| Option | Description | Selected |
|--------|-------------|----------|
| 有効化トグル+順序リスト | 「フォールバックを有効にする」ON/OFFトグルを既定OFFで表示、ONで順序リストが現れる | ✓ |
| リストが空なら自動的に無効 | 専用トグルは置かず、順序リストに1以上エントリがあれば有効とみなす | |

**User's choice:** 有効化トグル+順序リスト（推奨）
