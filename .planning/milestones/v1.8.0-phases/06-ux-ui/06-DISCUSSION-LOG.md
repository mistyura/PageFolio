## トースト対象エラーの線引き

### Q1: トースト化するエラーの選定方針

| Option | Description | Selected |
|--------|-------------|----------|

| Option | Description | Selected |
|--------|-------------|----------|

| 保存系＋印刷＋モデル一覧取得 | LLM 設定ダイアログのモデル一覧取得失敗も追加（ただし既にラベル通知あり） | |

### Q3: 再試行ボタンの動作

| Option | Description | Selected |
|--------|-------------|----------|

| 再試行＋代替アクション | 保存失敗には「別名で保存」ボタンも並べる等、操作別の複数導線 | |

### Q4: 再試行しても再び失敗した場合

## トーストの表示形態・挙動

### Q1: 実装方式

| Option | Description | Selected |
|--------|-------------|----------|
| メインウィンドウ内オーバーレイ（推奨） | place() で重ねる常駐 Frame。テーマ/フォント適用が容易・ウィンドウ移動に自然追従 | ✓ |
| Toplevel + overrideredirect | OS レベル浮遊ウィンドウ。ウィンドウ外にも出せるが挙動に癖 | |

### Q2: 表示位置

| Option | Description | Selected |
|--------|-------------|----------|
| 右下（推奨） | トーストの業界標準位置。操作導線と重ならない | ✓ |
| 上部（ヘッダー下） | _set_status と同じ視線先。ツールバー操作の邪魔になりやすい | |

### Q3: 複数エラー同時発生時

| Option | Description | Selected |
|--------|-------------|----------|
| 1件のみ表示（新しい方で置換・推奨） | スタック管理・レイアウト再計算不要で実装最小 | ✓ |
| 縦にスタック表示 | 将来対象を広げた場合に備えるが今回の少数対象では過剰設計 | |

### Q4: 消滅条件

| Option | Description | Selected |
|--------|-------------|----------|
| 同一操作の成功でも消す（推奨） | 別経路での再実行成功時もトーストを消し、古い失敗通知の残留を防ぐ | ✓ |
| ✕と再試行成功のみ | 実装最小だが別経路成功後も失敗トーストが残り続ける | |

## スクロール/フォント監査の基準と是正範囲

### Q1: スクロールの統一基準

| Option | Description | Selected |
|--------|-------------|----------|
| v1.7.2 llm_config パターンを正とする（推奨） | Canvas + Scrollbar + 高さクランプ + 下部ボタン固定 + マウスホイール対応の実績あるパターン | ✓ |
| 監査してから基準を策定 | 全箇所を比較して最良の形を基準化（確定が後ろへずれ込む） | |

### Q2: 是正の実装形態

| Option | Description | Selected |
|--------|-------------|----------|
| 不一致箇所の個別是正のみ（推奨） | 共通ヘルパー新設なし。回帰面が狭く「仕上げフェーズを軽く保つ」方針と整合 | ✓ |
| 共通ヘルパーを新設して移行 | make_scrollable_frame() 等へ移行。一貫性は強いがリグレッション面が広い | |

### Q3: フォント監査の是正範囲

| Option | Description | Selected |
|--------|-------------|----------|
| 数値ハードコードのみ是正（推奨） | font_size 設定に追従しない箇所だけ _font(delta) ベースへ。fs 連動済み・意図的固定は対象外 | ✓ |
| _font ヘルパーへ全面一本化 | fs 直算も含め統一。ui_builder の style 定義群を広く触ることになる | |

### Q4: 再発防止策

| Option | Description | Selected |
|--------|-------------|----------|
| フォントのみ回帰テスト化（推奨） | ソーススキャン型テスト追加（test_source_keyguard.py 前例踏襲・allowlist つき）。スクロールは監査記録のみ | ✓ |
| テスト化はしない | 監査は今回限り。SUMMARY・コミット記録のみ | |
