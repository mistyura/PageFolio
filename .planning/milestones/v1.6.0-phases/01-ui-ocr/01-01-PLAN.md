OCR 抽出画面（`OCRDialog`）の永続的な数値パラメータ UI を読み取り専用化し、編集導線を「⚙ LLM 設定…」（`LLMConfigDialog`）へ一元化する。これにより「OCR 画面と LLM 設定画面のどちらの値が効くか分からない」二重化（V16-UI-01）を解消する。読み取り専用表示は現在適用される値を見える状態で維持し（完全撤去ではなく D-01/D-04 の「読み取り専用化」を採用）、LLM 設定で値を変更・適用した直後に OCR 画面の表示も即時反映する（D-03）。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLMConfigDialog → app.settings → OCRDialog 表示 | 設定値がダイアログから settings dict を経て読み取り専用表示へ流れる（ローカル完結・外部入力なし） |
| app.settings → pagefolio_settings.json | `_save_settings` による永続化（既存経路・本フェーズで変更しない） |

> このフェーズは完全にローカルな Tkinter UI リファクタである。新規ネットワーク I/O・新規ファイル/資格情報処理・新規外部アタックサーフェスを一切導入しない。数値パラメータ Spinbox を読み取り専用化し、編集面を既存 LLMConfigDialog へ集約するのみ。

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | _apply_llm_settings の数値同期ログ | mitigate | 数値同期処理はログ出力を行わない。既存の `logger.error` は provider 再生成失敗時の例外メッセージのみで、API キー値や session key を出力しない（既存挙動を維持） |
| T-01-02 | Tampering | _SENSITIVE_KEYS 非永続化ガード | mitigate | 本タスクは数値パラメータ（非機密）のみを表示・同期し、api_key_var / _session_api_keys / _SENSITIVE_KEYS に一切触れない。読み取り専用化でセッション API キー欄（api_key_var）の編集可能性を変えない（D-06）ため、キーの settings 流入経路は新設されない |
| T-01-03 | Information Disclosure | model_combo / 読み取り専用 Spinbox 表示 | accept | 表示される数値は OCR 解像度・タイムアウト等の非機密設定値であり、ローカルデスクトップ画面上に元々表示されていた値。露出リスク増分なし |
</threat_model>
