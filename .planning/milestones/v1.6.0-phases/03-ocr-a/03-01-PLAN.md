<objective>
H1 ページ回転のプレビュー即時反映バグ（V16-QUAL-01）を「描画追加」ではなく「実機再現 → 真因特定 → 原因除去」で解消する。

Purpose: 回転ボタン押下後にプレビュー（現在ページ）と回転した選択中の全サムネイル（窓内可視範囲）が、再読込・ページ切替を待たず即時に回って見えるようにし、体感品質を底上げする。
Output: page_ops.py / viewer.py の最小修正 + tests/test_viewer.py の回転 w/h 単体テスト + UAT 目視確認手順。

RESEARCH 実測で PyMuPDF の `set_rotation` → `get_pixmap` は回転を即時反映する（90/270° で pixmap w/h 入替）ことが確認済み。よってバグは pixmap 描画層ではなく **Tk Canvas 層またはセレクション意味論**にある（A3）。本プランは viewer/page_ops に閉じ、OCR 系（Plan 02/03）と完全独立に並行実行できる。
</objective>

<locked_decisions>
本プランが従う確定事項（CONTEXT.md / RESEARCH.md 由来）:

- **D-01:** 現状は即時反映されないバグがある（ユーザー実機確認）。`_rotate_selected` は `_invalidate_thumb_cache(targets)` → `_refresh_all()`（→ `_show_preview` で `get_pixmap` 再描画）を呼んでいるのに即時反映されない。**原因のトレースと修正が中核**。修正は描画経路の追加ではなく原因除去。
- **D-02:** 即時反映の対象はプレビュー（現在ページ）＋回転した選択中の全サムネイル両方。`thumb_cache` は `targets` で無効化済みのためサムネイル側はキャッシュ無効化＋再描画が効く前提。窓正規化（Phase 2）により即時反映されるサムネイルは「現在の窓内に見えている範囲」に限られる（Pitfall 6・UAT/手順書に明記）。
- **D-03:** 担保は「検証可能な単位テスト＋手動 UAT」。`_render_preview_pixmap` が回転後の page を反映する（90/270° で w/h 入替）ことをテストし、最終的な見た目反映は手順書/UAT で目視確認。

**H1 真因の最有力仮説（実機 debug で確定する）:**
- (a) **セレクション意味論**: `_get_targets()`（app.py 195-196）は `selected_pages` があればそれを返し、無ければ `[current_page]` を返す。選択ページが `current_page` と異なる場合、回転後 `_show_preview` は `current_page`（= 回転対象外）を描画するため、プレビューが回らない「仕様上の体感バグ」。
- (b) **Canvas viewport/scrollregion**: 回転で w/h が入れ替わった際、scrollregion は再設定されるが xview/yview スクロール位置が旧寸法基準のまま残る（viewer.py 114-115）。
</locked_decisions>
