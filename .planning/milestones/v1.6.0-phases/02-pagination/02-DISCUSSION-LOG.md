## 表示件数の変更 UI と既定値

### 件数 UI の場所

| Option | Description | Selected |
|--------|-------------|----------|
| サムネイルフッターの Spinbox（ナビの隣） | 近接で発見性高い・その場で調整→即再描画 | ✓ |
| 設定ダイアログ（SettingsDialog） | UI すっきりだが変更の手間・発見しにくい | |

### 許容範囲（既定 20 は確定）

| Option | Description | Selected |
|--------|-------------|----------|
| 10〜100 | 下限 2 桁・上限 100 で描画負荷を抑える | ✓ |
| 5〜200 | 幅広だが 200 は低スペック PC で重い | |
| 任意（下限のみ） | 柔軟だが大きすぎる値でパフォーマンス劣化リスク | |

**User's choice:** フッター Spinbox（ナビの隣）、範囲 10〜100、既定 20、settings 永続化

## D&D・複数選択のクロス窓スコープ

### D&D 並べ替えのスコープ

| Option | Description | Selected |
|--------|-------------|----------|
| 表示中の窓内だけで並べ替え | ローカル→全ページ index 変換で move_page。安全・実装明確 | ✓ |
| クロス窓 D&D（別窓へ移動） | 表示外ページへのドロップ先指示が困難・高リスク | |

### 複数選択（selected_pages）の窓切替時の振る舞い

| Option | Description | Selected |
|--------|-------------|----------|
| 窓をまたいで保持（全ページ参照） | selected_pages は全ページ index 保持。成功基準 4 の意図 | ✓ |
| 窓切替時にクリア | 混乱は減るが窓またぎ一括操作ができない | |

### 「全選択/全解除」ボタンのスコープ

| Option | Description | Selected |
|--------|-------------|----------|
| 全ドキュメント（既存踏襲） | 全ページ選択/解除。選択保持方針と整合 | ✓ |
| 表示窓内のみ | 「見えているものだけ」直感的だが既存挙動から変わる | |

**User's choice:** 窓内 D&D、選択は窓またぎ保持、全選択は全ドキュメント
**Notes:** インデックス整合の核心。ローカル↔グローバル変換を 1 箇所に集約する方針。

## ページング適用のしきい値・端数・窓追従

### ページ数が表示件数以下のときの振る舞い

| Option | Description | Selected |
|--------|-------------|----------|
| 常にナビを表示（単一窓・ボタン無効） | 「1–8 / 全8」表示・ボタン disabled。UI が一定で予測可能 | ✓ |

### current_page が別窓に移ったときの表示窓

| Option | Description | Selected |
|--------|-------------|----------|
| current_page を含む窓へ自動追従 | 前/次ページ・D&D/削除後も選択/カレントが常に見える | ✓ |

**User's choice:** 常にナビ表示（単一窓はボタン無効）、current_page を含む窓へ自動追従

## Claude's Discretion

- 永続化キー名（候補 `thumb_page_size`）・窓状態属性名は実装裁量。
- Spinbox の即時反映タイミング・範囲ラベルの文言/区切り記号は既存規約に合わせて選択。
- D&D ゴースト/インジケータのローカル↔グローバル変換の実装詳細は裁量（「正しい全ページ位置へ移動」は必須）。

## Deferred Ideas

- サムネイル仮想化（本格的な遅延ロード基盤）→ 別タスク（本フェーズは窓区切りの最小実装）。
- クロス窓 D&D → 高リスクで不採用、将来別 UX として検討余地。
- ページ番号ジャンプ入力 / 窓ドロップダウン選択 → 最小実装では不採用。
- 回転プレビュー即時反映・OCR 堅牢性（プランA）→ Phase 3。
- OCR Markdown 整形・プロバイダ別プロンプト（プランC）→ Phase 4。
