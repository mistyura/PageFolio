## 入力欄の配置と見た目

| Option | Description | Selected |
|--------|-------------|----------|
| プロバイダ別セクション内（推奨） | 既存の claude/gemini/runpod セクションフレームに各 1 欄追加。選択中プロバイダのみ表示・既存 `_on_provider_change` を活用 | ✓ |
| 共通「APIキー」セクション | プロバイダ選択と独立に 3 欄常設。一括確認できるがダイアログが縦に伸びる | |

**User's choice:** プロバイダ別セクション内

| Option | Description | Selected |
|--------|-------------|----------|
| 常時マスク + 表示切替（推奨） | `show="*"` + 「👁 表示」トグルで入力ミス確認と肩越し防止を両立 | ✓ |
| 常時マスクのみ | 実装最小だが入力ミスを目視確認できない | |
| 平文表示 | 画面共有・スクリーンショット時に露出 | |

**User's choice:** 常時マスク + 表示切替

| Option | Description | Selected |
|--------|-------------|----------|
| 欄の下に小注記（推奨） | TEXT_SUB 色の小ラベルで「セッション限定・設定ファイル非保存」を表示。ja/en 両辞書へ追加 | ✓ |
| ラベルに簡潔に組み込む | 行数は増えないが意味が伝わりにくい | |
| 表示しない | 再起動後にキーが消えて驚く可能性 | |

**User's choice:** 欄の下に小注記

## 入力値のライフサイクル

| Option | Description | Selected |
|--------|-------------|----------|
| OK（適用）押下時（推奨） | `_apply` で `app._session_api_keys` へ直接格納。`llm_settings` dict には含めず T-05-12 ガードを構造的に維持。キャンセルで反映されない | ✓ |
| 入力即時反映 | KeyRelease ごとに反映。途中入力が残る・キャンセルで戻らない | |

**User's choice:** OK（適用）押下時

| Option | Description | Selected |
|--------|-------------|----------|
| マスク付きでプリフィル（推奨） | `_session_api_keys` の値を欄に入れて表示（マスク済み）。設定済みが一目で分かり上書き・削除も自然 | ✓ |
| 空欄 + 設定済みプレースホルダ | キー値が UI に乗らないが「空のまま OK」の意味が曖昧になる | |

**User's choice:** マスク付きでプリフィル

| Option | Description | Selected |
|--------|-------------|----------|
| 空にして OK で削除（推奨） | プリフィル方式と対になる自然な削除。追加ボタン不要 | ✓ |
| 専用「クリア」ボタン | 意図は明示的だが 1 欄に 2 ボタンになる | |

**User's choice:** 空にして OK で削除

## 環境変数との関係の見せ方

| Option | Description | Selected |
|--------|-------------|----------|

| 表示しない | シンプルだがどちらのキーが効いているか分からない | |
| 独立ステータス行 | 最も明示的だが行数が増える | |

| Option | Description | Selected |
|--------|-------------|----------|
| LLM設定への誘導に更新（推奨） | 「LLM設定ダイアログで APIキーを入力するか、環境変数 X を設定してください」へ更新（ja/en） | ✓ |
| 現行文言を維持 | 差分ゼロだが新導線に気づけない | |

**User's choice:** LLM設定への誘導に更新

| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** 文言での誘導のみ

## モデル取得ボタンのキー解決

| Option | Description | Selected |
|--------|-------------|----------|
| 入力欄のライブ値を最優先（推奨） | ダイアログ入力欄の現在値（OK 前でも）→ 環境変数。入力直後に取得＝疎通確認になる | ✓ |
| 適用済みセッションキー→環境変数 | OCR 実行と完全同一だが「OK で閉じて開き直す」手間が発生 | |
| 現行維持（環境変数のみ） | 差分ゼロだが入力キーだけのユーザーは静的一覧に限定される | |

**User's choice:** 入力欄のライブ値を最優先

| Option | Description | Selected |
|--------|-------------|----------|
| 静的リスト + ヒント表示（推奨） | 静的推奨モデルフォールバック（D-08 系）維持 + 「APIキー未設定のため推奨モデル一覧を表示中」ヒント | ✓ |
| 現行のサイレントフォールバック維持 | 差分ゼロだが取得できていないことに気づかない | |

**User's choice:** 静的リスト + ヒント表示
