<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ユーザー入力（Entry） → app._session_api_keys | 機密値（APIキー平文）がプロセスメモリ内 dict へ格納される。永続化層（settings.json）には渡さない境界 |
| LLMConfigDialog._apply → on_apply(llm_settings) → _save_settings | llm_settings dict が settings へ update され JSON 保存される。APIキーはこの境界を越えてはならない |
| 入力欄ライブ値 → _refresh_*_models → 外部 API（list_models） | ライブ入力キーが https で外部へ送られる（疎通確認）。キー値をログ/settings へ残さない |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-1-01 | Information Disclosure | llm_config.py::_apply → settings.json | high | mitigate | APIキーは _session_api_keys（メモリ dict）へのみ格納し llm_settings dict に含めない（D-04）。最後の砦の _SENSITIVE_KEYS ガードは変更せず維持。Task 3 の ApiKeyNotInSettings テストで回帰防止 |
| T-1-02 | Information Disclosure | _refresh_*_models 例外ログ・注記表示 | medium | mitigate | モデル取得失敗ログは provider 名のみ（既存 llm_model_fetch_failed 文言）・キー値は非出力。注記/ヒント文言も env_var 名のみでキー値を含めない。マスク show="*" が既定表示（D-02） |
| T-1-03 | Tampering | _apply のスロット格納 | medium | mitigate | provider キーを明示指定して _session_api_keys[provider] へ格納（runpod→runpod）。Task 3 の RunpodSessionKeySlot テストで claude スロット誤格納を回帰防止 |
| T-1-05 | Tampering | 2経路配線の非対称 | medium | mitigate | SettingsDialog 経由・OCRDialog 経由の双方へ同一 dict 参照を配線（複製禁止・Pitfall 5）。片系のみ配線による保存漏れを防ぐ |
| T-1-SC | Tampering | pip インストール | low | accept | 新規パッケージ導入なし（RESEARCH.md「Package Legitimacy Audit: 該当なし」）・受容 |
