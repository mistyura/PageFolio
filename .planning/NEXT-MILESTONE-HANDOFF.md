## 1. OAuth 接続に関する結論（確定事項）

**Claude / Gemini ともに、ユーザー個人アカウント枠での OAuth 接続は実装しない（見送り）。**

| 項目 | 確認結果 |
|------|----------|
| Claude 認証 | `x-api-key` ヘッダー方式のみ（`ocr_providers.py:344-348`）。Messages API は公式に API キー認証のみで、サードパーティアプリ向け OAuth ルートなし |
| Gemini 認証 | `x-goog-api-key` ヘッダー方式（`ocr_providers.py:553`）。OAuth 化には Vertex AI 移行が必要で重い |
| キー解決 | 環境変数優先 → セッションキー（`ocr.py:_resolve_api_key`）。`_SENSITIVE_KEYS` で `settings.json` 非保存 |
| 見送り理由 | ① 正規 API が非対応 ② デスクトップ配布バイナリに client secret を安全に埋め込めない ③ PageFolio はサーバを持たないローカル完結アプリでバックエンドプロキシ方式と相性が悪い |

> 改善するなら OAuth ではなく「キー入力 UX 改善（後述 M5）」で対応する。

## 4. 開発シナリオ（着手パッケージ案）

次マイルストーンの「目玉（テーマ）」に応じて、以下のいずれかを選んで着手する想定。

### プランA：堅実・クオリティ向上
- **対象**: H2 + H5 + H1 + M1
- **狙い**: 低難易度のリスク解消を先に終え、体感品質（回転プレビュー・エラー親切さ）を底上げ。ビルドクオリティを 1 段階上げる。

### プランB：機能拡張・PDF 強化
- **対象**: H4 + M2
- **狙い**: 暗号化 PDF の穴を埋め、ローカル履歴保存で「お仕事ツール」としての実用性を上げる。

### プランC：AI インテリジェンス強化
- **対象**: M4 + M3
- **狙い**: モデル別の出力特性（Claude=XML タグ / Gemini=明示指示）を引き出し、整形表示で読みやすさを上げる。
